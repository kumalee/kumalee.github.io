.editorconfig

  1. What is EditorConfig?

    EditorConfig helps developers define and maintain consistent coding styles between different editors and IDEs. The EditorConfig project consists of a file format for defining coding styles and a collection of text editor plugins that enable editors to read the file format and adhere to defined styles. EditorConfig files are easily readable and they work nicely with version control systems.

    简单的说呢，.editorconfig 就是用来定义和保持不同开发者、IDE、平台之间的代码风格的。

.gitignore

  1. What is .gitignore?

    If you create a file in your repository named .gitignore, Git uses it to determine which files and directories to ignore, before you make a commit. A .gitignore file should be committed into your repository, in order to share the ignore rules with any other users that clone the repository. See more https://help.github.com/articles/ignoring-files/
    简单来说呢，.gitignore 文件就是用来定义哪些文件/目录不需要被提交到项目的 Git 库中，为了
    和项目的协作开发者分享这个规则，应该把 .gitignore 文件本身添加到项目的 Git 库中去。
